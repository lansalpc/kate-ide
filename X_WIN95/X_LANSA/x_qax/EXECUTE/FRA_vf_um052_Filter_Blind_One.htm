Quoi ? Cet assistant CRUD produit le code pour un filtre d'objet métier caché.   
Qui ? Nouveaux développeurs 
Où ? Plateformes navigateur Web (génère du code navigateur Visual LANSA (RDMLX))
Détails ? Les assistants CRUD ont été conçus pour vous aider à développer rapidement les filtres et les gestionnaires d'action nécessaires à la Création/Lecture/Modification/Suppression des informations (CRUD) stockées dans le code commun ou les tables de définition du système. Ces types de tables existent dans de nombreuses applications métier (Sociétés, monnaies, codes postaux, départements, etc...) et ont en général peu de lignes et une faible fréquence de mise à jour. Les assistants CRUD accélèrent le processus de création des applications de maintenance pour ce type de tables. Le cycle de développement recommandé avec les assistants CRUD ressemble à ceci :
  • Définir une application Tables de Codes (par exemple)
  • Pour chaque code commun ou table de définition système :

    => Définir un objet métier (exemple : Devises, codes postaux).

    => Définir un seul filtre.

    => Activer seulement les actions Nouveau, Copier, Détails et Supprimer.

    => Répéter pour les plateformes Windows et Navigateur (suivant les besoins):

    • Créer un programme critère de recherche pour l'objet métier avec cet assistant. L'intégrer au cadre d'application.
    • Créer un seul programme gestionnaire d'action avec l'assistant de gestionnaire d'action CRUD. Intégrer le gestionnaire d'action pour chaque action Nouveau, Copier, Détails et Supprimer (i.e. le même programme est utilisé comme gestionnaire pour les quatre commandes).

Cliquez le bouton Générer le code pour générer le code du critère de recherche.

 
Entrez un nom et une description et cliquer sur le bouton Créer pour créer directement un nouveau réutilisable à partir du code généré,

ou,
