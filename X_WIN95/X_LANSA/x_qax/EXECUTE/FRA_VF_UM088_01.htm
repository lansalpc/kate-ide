Quelles actions les utilisateurs peuvent-ils effectuer avec leurs "objets métier" ?

Etape 2. Entrer les noms de toutes les actions possibles ci-dessous : (séparés par des virgules)

Les conceptions Windows utilisent une approche Objet->Action.
- (i.e. vous choisissez l'objet à manipuler, puis ce que vous voulez en faire.)
Les actions devraient être décrites avec les mots de l'utilisateur, pas de l'informaticien.

Utilisez des mots concis pour décrire les "actions", car l'objet manipulé est déjà connu.
Si vous choisissez un objet dans MS-Powerpoint, quand vous utilisez le clic droit, il y a une option de menu très simple "Copier".
- Il n'est pas écrit "Copier cette boite de texte dans le presse-papier".
De simples verbes sont souvent utilisés pour décrire les actions.
- (Copier, Nouveau, Modifier, Imprimer, Approuver, Transférer, Répondre, Renouveler)
Des noms courts sont aussi utilisés pour définir ce qui est directement lié à l'objet métier.
- (Détails, Historique, Charges, Demandes, Documents attachés, Calendrier, Contacts, Documents, Dépenses)
Par exemple : s'il existe un objet "métier" appelé Clients, vous pouvez utiliser pour chaque Client :
  Modifier, Imprimer, Supprimer, Comptes, Dernières Transactions, Correspondance, Vérifier.
